## 一、为什么需要大模型

### 传统 NLP 的痛点

以前做 NLP 项目，每个任务都要单独搞一套：

```
任务 A（情感分析）→ 标注数据 → 训练 BERT → 部署上线
任务 B（文本分类）→ 标注数据 → 训练 BERT → 部署上线
任务 C（实体识别）→ 标注数据 → 训练 BERT → 部署上线
```

**问题**：
- 每个模型单独维护，累
- 新任务要从零开始，慢
- 标注数据成本高，贵

### 大模型范式

现在只需要一个通用底座：

```
        通用大模型（如 LLaMA-3-8B）
              │
    ┌─────────┼─────────┐
    │         │         │
  LoRA-A    LoRA-B    LoRA-C
 (情感)    (分类)    (实体)
```

**优势**：
- ✅ 一个底座，多次复用
- ✅ 新任务只需少量数据
- ✅ 统一升级，维护简单

## 二、实战：成本对比

### 案例：电商公司 10 个 NLP 任务

**传统方案（10 个 BERT 模型）**

```bash
# 训练成本
BERT-base × 10 = $5,000

# 数据标注
10 任务 × $2,000 = $20,000

# 部署维护
10 模型 × $300/月 × 12 月 = $36,000

总计（首年）：$61,000
```

**大模型方案（1 个底座 + LoRA）**

```bash
# 底座（复用开源 LLaMA）
$0

# 数据标注（少量样本）
10 任务 × $500 = $5,000

# LoRA 微调
10 × $100 = $1,000

# 部署维护
1 模型 × $1,000/月 × 12 月 = $12,000

总计（首年）：$18,000
```

**节省：70%** 💰

## 三、什么时候用大模型？

### ✅ 适合大模型的场景

| 场景 | 原因 | 推荐模型 |
|------|------|---------|
| 复杂推理 | 需要多步思考 | 70B+ |
| 开放域问答 | 知识面要广 | 30B+ |
| 少样本学习 | 标注数据少 | 13B+ |
| 代码生成 | 理解逻辑结构 | 30B+ |

### ✅ 适合小模型的场景

| 场景 | 原因 | 推荐模型 |
|------|------|---------|
| 单一任务 | 数据充足 | BERT/RoBERTa |
| 延迟<50ms | 实时性要求高 | 7B INT4 |
| 高并发 | QPS>100 | 7B 量化 |
| 本地部署 | 隐私要求 | 7B-13B |

## 四、快速上手：选模型决策树

```bash
# 第一步：是否需要复杂推理？
if 需要推理:
    选择 70B+ 模型（GPT-4/Claude/LLaMA-3-70B）
else:
    继续下一步

# 第二步：延迟要求？
if 延迟 < 50ms:
    选择 7B 模型 + INT4 量化
elif 延迟 < 200ms:
    选择 13B-30B 模型
else:
    选择 70B+ 模型

# 第三步：预算？
if 预算低:
    开源模型（LLaMA/Qwen）+ 自建
elif 预算中等:
    云服务（DeepSeek/Moonshot）
else:
    商业 API（GPT-4/Claude）

# 第四步：数据隐私？
if 必须本地:
    开源模型 + 本地部署
else:
    商业 API
```

## 六、动手实践

### 1. 本地部署 7B 模型

```bash
# 安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 下载 LLaMA-3-8B
ollama pull llama3

# 运行
ollama run llama3 "你好，请介绍一下自己"
```

### 2. 调用 API

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://api.deepseek.com",
    api_key="你的 API_KEY"
)

response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[
        {"role": "user", "content": "你好"}
    ]
)

print(response.choices[0].message.content)
```

## 七、常见问题

### Q1：7B 模型够用吗？

**答**：对于 80% 的企业场景，7B 量化后足够。只有复杂推理才需要 70B+。

### Q2：本地部署还是用 API？

**答**：
- 数据敏感 → 本地
- 追求效果 → API
- 高并发 → 本地 + API 混合

### Q3：量化会损失多少精度？

**答**：INT4 量化通常损失 1-3% 精度，但速度提升 2-4 倍，显存减半。
