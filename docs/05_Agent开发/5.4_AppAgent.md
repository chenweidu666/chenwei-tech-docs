# 5.4. AppAgent 与 Android 端自动化

> **导航**：[← 5.3 Dify 工作流平台](5.3_Dify工作流平台.md) · [5.2 MCP 协议](5.2_MCP协议.md)

> **本章导读**：**AppAgent** 一类方案把移动端 UI 自动化抽象成 **「看屏幕 → 理解界面 → 下发操作」** 的闭环，依赖 **ADB** 与 **多模态大模型**。本节覆盖架构、环境、稳定性与自检，便于与端侧部署（[3.3 端侧部署](../03_部署与推理/3.3_端侧部署.md)）对照。

> **关键词**：AppAgent · ADB · 视觉理解 · GUI Agent · 热更新 · 自检

---

## 5.4.1. AppAgent 是什么

- **目标**：在 **无需应用提供官方 API** 的前提下，通过 **界面截图 + 自然语言任务**，完成安装、点击、输入、滑动等 **GUI 级自动化**。  
- **典型循环**：  
  1. **感知**：ADB 截取当前屏幕（可配合 XML 布局 dump 作辅助，视实现而定）。  
  2. **理解**：多模态模型识别当前页面、控件语义与任务进度。  
  3. **规划**：输出下一步操作（坐标/控件描述 → 映射为 ADB 指令）。  
  4. **执行**：`input tap`、`input swipe`、`input text` 等。  
  5. **验证**：再次截图或检查界面变化，失败则重试或改策略。  
- **与 RPA/传统脚本的区别**：弱依赖固定坐标与脚本分支，**泛化**更强，但 **时延与成本**更高，需工程上约束步数与模型调用频率。

---

## 5.4.2. 核心架构

### 5.4.2.1. 感知层（ADB 截图）

- **基础命令**：`adb exec-out screencap -p` 或 `adb shell screencap`；注意分辨率、横竖屏与 **隐私弹窗**（截图可能黑屏）。  
- **性能**：降低截图频率、按需截屏（状态变化后再截）；可缩小分辨率供模型分析以省 token。  
- **辅助信息（可选）**：`uiautomator dump` 获取 **可访问性树**，与像素信息互补，提高控件定位稳定性。

### 5.4.2.2. 理解层（LLM / 多模态模型）

- **输入**：当前截图 + 任务描述 + 简短历史（上一步操作与结果摘要）。  
- **输出**：结构化 **动作 JSON**（类型、坐标或控件 id、文本内容、等待时间等），便于程序解析而非自由文本。  
- **提示词**：明确 **安全边界**（不碰支付密码、不越权）、**失败时如何报告**、**何时认为任务完成**。  
- **模型选型**：云端多模态（能力强）vs 端侧小模型（延迟低）；可 **大小模型级联** 做意图分类。

### 5.4.2.3. 执行层（ADB 操作指令）

- **点击/滑动**：`input tap x y`、`input swipe x1 y1 x2 y2 duration`。  
- **文本**：`input text`（注意转义与输入法）；复杂场景可用 **ADB Keyboard** 或剪贴板粘贴。  
- **系统键**：`KEYCODE_BACK`、`HOME` 等。  
- **稳定性**：操作后 **sleep** 适度等待动画；连续失败时 **回到已知首页** 再重试。

## 5.4.3. 环境搭建

| 步骤 | 说明 |
|------|------|
| ADB | 安装 Android SDK platform-tools；`adb devices` 能看到设备 |
| 开发者选项 | 开启 **USB 调试**；部分机型需 **USB 调试（安全设置）** 才能模拟输入 |
| 连接 | USB 或 `adb tcpip 5555` 无线调试；注意防火墙与稳定性 |
| Python 环境 | 脚本驱动截图、调模型 API、解析 JSON、循环状态机 |
| LLM API | 配置 Qwen-VL、GPT-4V 等；设置 **超时、重试、费用上限** |

**建议**：单独仓库维护 **设备配置**（分辨率、dpi、厂商差异），用配置表减少硬编码坐标。

---

## 5.4.4. 热更新架构

- **动机**：提示词、工具列表、动作解析规则迭代快，避免每次改代码发版。  
- **做法**：  
  - 远程拉取 **YAML/JSON**（系统提示、允许的动作类型、各 App 的包名与白名单页面）；  
  - 本地 **缓存 + 版本号**，失败回退到内置默认；  
  - 关键策略变更打 **审计日志**。  
- **风险**：远程配置被篡改 → 必须 **HTTPS + 签名或校验和**；生产环境建议 **只读 CDN + 审批发布**。

## 5.4.5. 实战：内容平台「自动发布」流程（抽象）

> 以「打开 App → 登录态已存在 → 进入发布页 → 填标题正文 → 上传图片 → 提交」为例，步骤因 App 改版会经常变化，**语义层稳定、选择器层要可换**。

1. **任务分解**：导航到发布入口 → 填写表单 → 处理弹窗（权限、活动）→ 提交 → 确认成功页。  
2. **每步**：截图 → 模型判断当前页是否预期 → 输出动作 → ADB 执行 → 验证。  
3. **异常**：网络错误、风控验证码、登录失效 → 定义 **明确终止码** 并通知人工。  
4. **合规**：仅用于 **自己账号、授权设备** 的自动化；遵守平台 ToS，避免批量灰产场景。

## 5.4.6. 自检模式（`--self-debug` 类）

- **目的**：自动化回归「截图 → 解析 → 动作」链路是否通畅，不依赖完整业务任务。  
- **做法**：  
  - 固定测试页或测试 App，执行 **黄金路径** 若干步；  
  - 比对截图哈希或关键区域 OCR 是否与期望一致；  
  - 记录每步 **模型原始输出、解析后指令、执行耗时**。  
- **接入 CI**：真机农场或长期挂测机；失败发钉钉/飞书/邮件。

## 5.4.7. 踩坑记录

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 截图黑屏 | 金融/支付类防截屏 | 换用无障碍树 + 有限操作，或仅非敏感页自动化 |
| 坐标偏移 | 分辨率、异形屏、悬浮窗 | 归一化坐标或优先走控件 id |
| 输入失败 | 特殊字符、输入法抢占 | ADB Keyboard、剪贴板 |
| 模型乱跳页 | 提示词不够约束、历史过长 | 缩短上下文、强制每步只选一个动作 |
| 成本暴涨 | 每步高分辨率原图 | 降采样、关键帧截屏、状态未变不调用模型 |
| 合规风险 | 批量操控、爬取用户数据 | 产品边界与法务评估前置 |
