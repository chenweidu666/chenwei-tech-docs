## 二、关键技术演进

### MoE（Mixture of Experts）

**核心思想**：总参数大，但每步只激活部分。

```
Mixtral 8×7B：
- 总参数：47B
- 激活参数：13B（每步只用 2 个专家）
- 效果：接近 47B，速度接近 13B
```

### GQA（Grouped-Query Attention）

**核心思想**：减少 K/V 头数，降低显存占用。

| 机制 | Q 头 | K 头 | V 头 | KV Cache |
|------|-----|-----|-----|---------|
| MHA | 64 | 64 | 64 | 100% |
| MQA | 64 | 1 | 1 | 3% |
| GQA | 64 | 8 | 8 | 25% |

### 量化技术

| 精度 | 显存 | 速度 | 精度损失 |
|------|------|------|---------|
| FP16 | 100% | 1x | 0% |
| INT8 | 50% | 1.5x | 0.5% |
| INT4 | 25% | 2.5x | 2% |
| INT2 | 12.5% | 4x | 5% |

## 三、学习路线

### 入门（1-2 周）

```
1. 理解 Transformer 架构
   ├─ 自注意力机制
   ├─ 位置编码
   └─ 编码器/解码器

2. 会用 HuggingFace
   ├─ 加载预训练模型
   ├─ 简单微调
   └─ 推理部署

3. 了解主流模型
   ├─ LLaMA 系列
   ├─ Qwen 系列
   └─ DeepSeek 系列
```

### 进阶（1-2 月）

```
1. 微调技术
   ├─ 全量微调
   ├─ LoRA/QLoRA
   └─ RLHF/DPO

2. 部署优化
   ├─ vLLM/TGI
   ├─ 量化（INT4/FP8）
   └─ 推理加速

3. RAG 与 Agent
   ├─ 向量数据库
   ├─ LangChain
   └─ Agent 工作流
```

### 高级（3-6 月）

```
1. 预训练
   ├─ 数据清洗
   ├─ 分布式训练
   └─ 训练稳定性

2. 模型架构创新
   ├─ 注意力改进
   ├─ MoE 设计
   └─ 长上下文优化

3. 垂直领域应用
   ├─ 医疗/法律/金融
   ├─ 代码生成
   └─ 多模态
```

## 七、常见问题

### Q1：非科班能学大模型吗？

**答**：可以！建议路线：
1. 先学 Python 基础
2. 再学 PyTorch
3. 然后学 Transformer
4. 最后学大模型应用

### Q2：需要多少数学基础？

**答**：
- 线性代数（矩阵运算）
- 概率统计（基础概念）
- 微积分（导数/梯度）

高中数学 + 大一高数足够入门。

### Q3：如何保持学习动力？

**答**：
- 设定小目标（每周完成一个小项目）
- 加入学习社群（互相监督）
- 写技术博客（输出倒逼输入）
- 参与开源（获得反馈）
