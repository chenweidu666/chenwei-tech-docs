## OpenClaw 常见问题

### 问题 1：无法启动

**错误**：
```
Error: Cannot find module 'openclaw'
```

**解决**：
```bash
# 重新安装
npm uninstall -g openclaw
npm install -g @qingchencloud/openclaw-zh --registry https://registry.npmmirror.com

# 验证
openclaw --version
```

---

### 问题 2：配置文件损坏

**错误**：
```
Error: Invalid JSON in config file
```

**解决**：
```bash
# 备份损坏的配置
cp ~/.openclaw/config.json ~/.openclaw/config.json.bak

# 重新生成配置
openclaw init --force

# 恢复自定义配置
vim ~/.openclaw/config.json
```

---

### 问题 3：内存不足

**错误**：
```
<--- Last few GCs --->
FATAL ERROR: Ineffective mark-compacts near heap limit
```

**解决**：
```bash
# 增加 Node.js 内存限制
export NODE_OPTIONS="--max-old-space-size=4096"

# 永久生效
echo 'export NODE_OPTIONS="--max-old-space-size=4096"' >> ~/.bashrc
source ~/.bashrc
```

---

### 问题 4：会话丢失

**症状**：对话记录消失

**解决**：
```bash
# 检查会话目录
ls -la ~/.openclaw/agents/main/sessions/

# 恢复备份
cp ~/.openclaw/agents/main/sessions/*.bak ~/.openclaw/agents/main/sessions/

# 重启 Gateway
openclaw gateway restart
```

## Gateway 问题

### 问题 1：端口被占用

**错误**：
```
Error: Port 18789 is already in use
```

**解决**：
```bash
# 查找占用进程
lsof -i :18789

# 杀死进程
kill -9 $(lsof -t -i :18789)

# 或修改端口
vim ~/.openclaw/config.json
# 修改 "port": 18790

# 重启
openclaw gateway restart
```

---

tail -f ~/.openclaw/agents/main/logs/gateway.log

## Git 同步问题

### 问题 1：SSH 认证失败

**错误**：
```
fatal: Could not read from remote repository.
Permission denied (publickey).
```

**解决**：
```bash
# 检查 SSH Key
ls -la ~/.ssh/github_key

# 测试连接
ssh -i ~/.ssh/github_key -T git@github.com

# 配置 Git
git config --global core.sshCommand "ssh -i ~/.ssh/github_key -o IdentitiesOnly=yes"

# 检查公钥是否添加到 GitHub
# https://github.com/settings/keys
```

---

### 问题 2：Git 冲突

**错误**：
```
CONFLICT (content): Merge conflict in MEMORY.md
```

**解决**：
```bash
cd /root/.openclaw/workspace/GateClaw-Bot

# 查看冲突
git diff MEMORY.md

# 手动编辑解决
vim MEMORY.md

# 标记解决
git add MEMORY.md

# 继续 rebase
git rebase --continue

# 或放弃 rebase
git rebase --abort

# 推送
git push
```

---

### 问题 3：推送失败

**错误**：
```
! [rejected] master -> master (fetch first)
```

**解决**：
```bash
# 先拉取
git pull --rebase origin master

# 如果有冲突，解决后
git add .
git commit -m "resolve conflict"
git push

# 或强制推送（慎用）
git push -f origin master
```
