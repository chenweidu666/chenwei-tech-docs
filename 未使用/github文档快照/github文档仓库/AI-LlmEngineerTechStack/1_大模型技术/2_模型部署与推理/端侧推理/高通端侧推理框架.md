# 高通端侧推理框架

## 概述

目前没有找到名为 "damox" 的高通官方端侧推理框架。可能是名称误记或内部代号。以下是高通当前主流的端侧推理框架和技术栈。

## 高通主流端侧推理框架

### 1. QNN SDK (Qualcomm AI Engine Direct SDK)

- **官方 SDK**，用于将模型转换/优化/部署到 Hexagon NPU
- 支持 ONNX、TensorFlow、PyTorch 等格式
- 提供 NPU 加速的执行提供器（Execution Provider）
- 官方文档：https://microsoft.github.io/Olive/0.6.0/features/passes/qnn.html

### 2. Qualcomm AI Hub / Workbench

- **在线平台**，支持模型硬件感知优化
- 可在指定 Qualcomm 设备上编译、性能分析和部署评估
- 支持多种模型格式转换
- 官方平台：https://app.aihub.qualcomm.com

### 3. TensorFlow Lite + QNN Delegate

- 在 Android/Snapdragon 上使用 QNN Delegate
- 将 TensorFlow Lite 模型委托给 NPU 或 Hexagon DSP 执行
- 提供硬件加速支持
- 文档：https://docs.edgeimpulse.com/tutorials/topics/android/qnn-acceleration

### 4. LiteRT + QNN Accelerator（较新）

- Google LiteRT 与高通合作的加速器
- 统一 Android 端 NPU 推理流程
- 提供更简单且高性能的部署方式
- 官方博客：https://developers.googleblog.com/unlocking-peak-performance-on-qualcomm-npu-with-litert/

### 5. QAIRT (Qualcomm AI Runtime)

- AI 推理运行时的一部分
- 支持模型在 AI Engine/NPU 上高效执行
- 提供版本控制和兼容性层
- 文档：https://workbench.aihub.qualcomm.com/docs/hub/frameworks.html

## 关于 "damox" 的可能情况

1. **可能是 "QNN" 的误读或听写错误**：比如 "dee-nox / que-nox / damox" 之类
2. **可能是内部项目代号**：非公开或内部研发项目
3. **可能是第三方框架**：开源框架或第三方库的非官方命名
4. **可能是量化相关术语**：Quantization-Aware DNN / Quantized Model Deployment 的简写

## 总结

**Qualcomm 用来做端侧推理最正式、最主流的框架是 "Qualcomm AI Engine Direct (QNN)"**，配合 Hexagon NPU + TF-Lite Delegate 或 ONNX Runtime 的 QNN Execution Provider。最新的 LiteRT 与 QNN Accelerator 也越来越被强调。
