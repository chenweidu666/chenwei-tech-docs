## 1.1.1. 为什么需要大模型

- **统一底座，多任务复用**：传统 NLP 往往「一任务一模型」。大模型在超大规模语料上做自监督预训练后，可通过 SFT、RAG、工具调用等方式适配多种任务，减少重复造轮子。
- **规模带来的能力跃迁**：当参数量、数据量与算力超过一定阈值后，会出现 **涌现能力**（如复杂推理、多步指令遵循、少样本学习），这一现象与「缩放定律」（Scaling Laws）的实证研究一致——在合理配方下，损失随规模可预测下降。
- **工程上的取舍**：更大模型意味着更高显存、更长延迟与更高成本，因此实际系统要在 **能力 / 时延 / 成本** 之间做权衡；理解架构（如 GQA、MoE）有助于在不线性增加算力的情况下缓解瓶颈。

## 1.1.2. Transformer 架构详解

现代 LLM 几乎清一色基于 **Transformer**；差异主要在：仅解码器（Decoder-only）、编码器-解码器（Encoder-Decoder）、是否使用 MoE 等。

### 1.1.2.1. Self-Attention 原理

- **要解决的问题**：对长度为 \(n\) 的序列，让每个位置都能「看到」其它位置的信息，并学习 **谁该关注谁**。
- **核心计算（缩放点积注意力）**：将输入映射为 Query \(Q\)、Key \(K\)、Value \(V\)，注意力权重由 \(QK^\top\) 经缩放与 softmax 得到，再对 \(V\) 加权求和。直观理解：**用 Query 去匹配 Key，用权重去聚合 Value**。
- **因果掩码（Causal Mask）**：自回归生成时，位置 \(i\) 只能看见 \(\le i\) 的位置，保证训练和推理一致；BERT 类双向模型则不用因果掩码。

### 1.1.2.2. Multi-Head Attention

- **多头**：将 \(Q,K,V\) 拆成多组子空间并行计算，再拼接投影。意义类似 CNN 多通道：不同头可捕捉不同关系（语法、指代、长距离依赖等）。
- **与推理的关系**：多头使每层状态更丰富，但也增加 **KV Cache** 的存储量（每层每头都要缓存）。

### 1.1.2.3. FFN 与 MoE

- **FFN（前馈层）**：通常对每个 token 独立做两层 MLP（如维度扩张再压回），是参数量的大头之一；常见激活包括 GeLU、**SwiGLU**（LLaMA 系常用）。
- **MoE（混合专家）**：将部分 FFN 换成「多专家 + 门控」，每个 token 只激活少数专家，从而在 **总参数量很大** 的同时控制 **单次前向的计算量**。适合超大规模模型（如部分 DeepSeek、Mixtral 路线）。

## 1.1.3. 主流模型架构对比

| 模型 | 架构特点 | 参数量（量级） |
|------|----------|----------------|
| LLaMA | Decoder-only，**RoPE** 位置编码，**RMSNorm**，SwiGLU；后期多用 **GQA** | 7B–70B+ |
| Qwen | Decoder-only，**GQA**，中文与代码数据友好，生态工具链全 | 0.5B–72B+ |
| DeepSeek | 密集 + **MoE** 等多条产品线，推理与代码场景强 | 7B–671B 级 |

**阅读建议**：选型时不仅看参数量，还要看 **上下文长度、许可证、工具链（vLLM、量化、Agent）** 与业务语言分布。

## 1.1.4. KV Cache 原理与优化

- **是什么**：自回归生成第 \(t+1\) 个 token 时，前面各层的 **K、V** 对已生成前缀不变，可缓存下来，避免每步重复计算整段历史——这就是 **KV Cache**。
- **显存占用直觉**：大致与 **层数 × 序列长度 × 隐藏维度 × 精度 × 头数/分组数** 成正比；上下文从 4K 拉到 128K 时，KV 往往成为首要瓶颈。
- **常见优化方向**：
  - **GQA / MQA**：多查询注意力，减少 K/V 头数，降低 Cache 体积。
  - **PagedAttention（vLLM）**：把 KV 分页管理，减少碎片与浪费（详见 [3.1 vLLM 推理部署](../03_部署与推理/3.1_vLLM推理部署.md)）。
  - **量化 KV**：INT8/FP8 等降低单元素占用（需权衡精度）。

---

## 1.1.5. 推理加速关键技术

- **批处理与调度**：**Continuous Batching** 动态拼 batch，提高 GPU 利用率（vLLM 等）。
- **算子与图优化**：FlashAttention 系列降低注意力显存与 IO；融合内核减少 launch 开销。
- **投机解码（Speculative Decoding）**：用小模型草稿 + 大模型验证，换吞吐（实现复杂，需框架支持）。
- **权重量化**：W8A16、GPTQ/AWQ 等降低带宽压力（与 [3.2 量化](../03_部署与推理/3.2_INT量化.md) 衔接）。
