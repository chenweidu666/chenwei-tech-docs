> **时间**：2025.12 – 至今｜**性质**：个人项目  
> **开源**：[CineMaker-AI-Platform](https://github.com/chenweidu666/CineMaker-AI-Platform)（主仓库）· [CineMaker_Public](https://github.com/chenweidu666/CineMaker_Public)（公开镜像）

<h1 align="center">项目背景与目标</h1>

- **痛点**：短剧从剧本到成片链路长、角色/场景一致性难控，传统制作人力与周期成本高。  
- **目标**：将「资源准备 → 剧本 → 分镜 → 生图 → 视频 → 合成」收束为**可重复执行的产品化流水线**，在个人或小团队场景下做到 **1 人 + 约 1 小时 + 约 ¥20 量级 API 成本** 完成单集（参考体量：**约 13 镜头 / ~90 秒** 类 Vlog/短剧）。  
- **定位**：全栈 Web 产品（非脚本 Demo），默认 **单体 + SQLite** 开箱即用，便于 Docker 一键部署。

---

<h1 align="center">端到端工作流（五阶段）</h1>

与归档中《产品工作流》一致，核心阶段为：

1. **资源准备**：角色（含**多造型**）、场景库、道具库；为后续生成提供**可引用的设定与参考图**。  
2. **剧本创作**：章节与资源关联后，由大模型生成剧本（避免「凭空造角色」导致与资源库脱节）。  
3. **分镜拆分**：**三段式分镜**（首帧 / 过程 / 尾帧），便于控制镜头叙事与后续图生视频。  
4. **图片生成**：首帧、尾帧生成；支持**参考帧**衔接镜头；必要时走**图片微调/编辑**闭环。  
5. **视频与合成**：基于火山**首尾帧类视频模型**逐镜头生成，内置 FFmpeg **时间线/转场/合并**；成片精剪可导出至剪映等工具。

```mermaid
flowchart LR
  A[资源准备] --> B[剧本]
  B --> C[分镜]
  C --> D[首尾帧图]
  D --> E[逐镜视频]
  E --> F[合成成片]
```

<h1 align="center">技术架构（摘要）</h1>

摘自归档《技术架构》，便于与 [5.1 Agent 设计](../05_Agent开发/5.1_Agent设计与工作流.md) 对照：

| 层级 | 选型 | 说明 |
|------|------|------|
| 后端 | Go、Gin、GORM、**SQLite（WAL）** | 单二进制、接口 `/api/v1/*`；Handler + Service 分层，业务编排主要在 `application/services` |
| 前端 | Vue 3、TypeScript、Vite、Element Plus、Tailwind | SPA + Pinia |
| AI | **火山引擎** | 文本 Doubao；图片 Seedream / 编辑 Seededit；视频 Seedance；视觉理解等按产品配置 |
| 媒体 | **FFmpeg** | 合成、截帧、转码等 |
| 部署 | Docker / Compose；开发 **air** 热更 | 存储默认本地，**Storage 接口**可扩展对象存储（如 COS） |

> **说明**：上游文档以 **Go 服务层编排** 描述为主；若对外介绍中曾用「多 Agent」指**阶段化自动链路**（剧本/分镜/生图/视频等模块），与「单一框架名」不必强行绑定，以仓库实现为准。

<h1 align="center">一致性方案（面试/复盘重点）</h1>

- **参考图驱动**：角色三视图、场景空镜、道具设定图进入后续镜头生成上下文，减轻「每镜一张脸」。  
- **多造型（Appearances）**：同一角色多套造型，分镜绑定**具体造型**而非仅角色 ID。  
- **三段式分镜 + 首尾帧视频**：约束单镜时间与动作描述，配合参考帧降低**跨镜头漂移**。  
- **提示词工程**：项目内另有《提示词指南》，与火山 Seedream/Seedance 官方最佳实践对齐（详见归档）。

---

<h1 align="center">核心成果与指标（自述口径）</h1>

- **产能**：单集（约 13 镜 / ~90s）在熟练流程下 **~1 小时级** 跑通 AI 素材链路（精剪仍建议在剪映等完成）。  
- **成本**：同体量下 API 费用**约 ¥20 量级**（以火山现网计费为准，随模型/分辨率/时长波动）。  
- **工程化**：可配置 API、异步任务与媒体管线、可扩展存储；适合作为 **「多模态 + 工作流产品」** 案例写入简历与答辩。

---

<h1 align="center">踩坑记录（提纲）</h1>

- 参考图数量与顺序对 Seedream/Seedance 效果影响大，需模板化与回归用例。  
- 长链路任一步失败要有**可重试、可跳过、可局部重跑**，避免整集重来。  
- 费用与耗时随「分辨率、时长、重试次数」敏感，需在 UI/日志层**可观测**。

---

<h1 align="center">经验总结</h1>

- **产品型 Agent**：价值在「状态机 + 资源模型 + 可观测 + 成本约束」，而非单点提示词。  
- **一致性**优先靠**数据结构与参考图策略**，其次才是模型换大。  
- 与 [5.3 Dify](../05_Agent开发/5.3_Dify工作流平台.md) 对比：CineMaker 是**垂直领域全栈实现**，Dify 更偏通用编排平台；二者可对照讲清选型。
