# 1.2 数据处理与预训练

> **导航**：[← 上一节：1.1 大模型基础与架构](1.1_大模型基础与架构.md) · [下一章：2.1 SFT 监督微调 →](../02_训练与微调/2.1_SFT监督微调.md)

> **本章导读**：预训练决定模型的「世界知识与语言习惯」。数据怎么来、怎么洗、怎么切成 token、用什么目标函数训练，是后续一切微调与 RAG 的上限所在。

> **关键词**：数据清洗 · Tokenizer · 预训练目标 · 数据配比

---

## 一、预训练数据来源

- **通用网页与语料**：规模大、覆盖广，但噪声多、重复多，需严格清洗与去重。
- **书籍、论文、百科**：语言规范、事实密度高，有利于知识与推理基础；常通过授权语料或公开数据集引入。
- **代码与 Markdown**：提升代码生成、指令遵循与结构化文本能力；与工具链、Agent 场景强相关。
- **多语言配比**：按产品目标调整中英文等比例；过少会导致某语言「夹生」或能力偏科。
- **合规与版权**：商用需明确授权边界；个人笔记站与归档内容不宜直接混入预训练流水线。

---

## 二、数据清洗流程

### 2.1 去重（MinHash / SimHash）

- **目的**：重复段落会放大训练信号、浪费算力，并可能导致模型「背答案」式复述。
- **MinHash / LSH**：适合大规模近重复检测，将相似文档映射到同一桶，再精细比对。
- **SimHash**：得到固定长度指纹，汉明距离小的判为近似重复；实现简单，适合网页去重。
- **实践**：常分级——URL/文档级去重 → 段落或 n-gram 级去重，平衡精度与算力。

### 2.2 质量过滤

- **启发式规则**：长度、符号比例、乱码检测、语言识别（只保留目标语种）。
- **模型打分**：用小语言模型算 **困惑度（Perplexity）** 过滤异常低/高片段；或用 **质量分类器**（fastText、小 Transformer）打分。
- **去模板化**：导航栏、版权声明、Cookie 提示等网页模板会污染语料，可用 DOM 规则或聚类模板去除。

### 2.3 安全过滤

- **PII**：姓名、电话、证件等需脱敏或剔除，满足隐私合规。
- **有害内容**：暴力、违法、儿童安全等类别，常用关键词 + 分类模型 + 人工抽检多道闸。
- **可审计**：保留过滤规则版本与抽样集，便于复盘与迭代。

## 三、Tokenizer 设计

### 3.1 BPE / WordPiece / SentencePiece

- **BPE（Byte Pair Encoding）**：从字符起步，迭代合并高频子词对；**GPT / LLaMA 系**常用。
- **WordPiece**：类似 BERT，基于似然增量选择合并；偏子词级，OOV 少。
- **SentencePiece**：把空格也当符号，**不依赖预分词**，适合中日韩等无空格语言；许多多语言模型采用。
- **字节级 BPE**：词表可覆盖任意 UTF-8 序列，减少 UNK，但序列可能变长。

### 3.2 词表大小选择

- **常见范围**：32k–128k 量级；更大词表 → 同样文本 **token 数更少**，但 embedding 矩阵变大、 softmax 头更重。
- **权衡**：中文场景若词表过小，会拆成过多字节或子词，**有效上下文被压缩**；需在「序列长度、显存、推理速度」间实测。
- **与业务对齐**：若领域术语极多，可在继续预训练或微调阶段观察 OOV/切分是否异常，必要时考虑词表扩展（成本高，需谨慎）。

---

## 四、预训练目标

### 4.1 CLM（因果语言模型）

- **形式**：给定前缀，预测下一 token；**GPT、LLaMA、Qwen** 等生成式 LLM 的主目标。
- **特点**：天然适配 **自回归生成**；预训练好的模型经 SFT 即可对话。
- **损失**：对整段文本的 next-token 交叉熵（配合因果掩码）。

### 4.2 MLM（掩码语言模型）

- **形式**：随机 mask 部分 token，用双向上下文预测被掩码位置；**BERT** 系代表。
- **特点**：擅长 **编码与理解** 任务；不直接等价于自回归生成，往往需额外头或再训练做生成。
- **现状**：当前通用 LLM 主线以 Decoder-only + CLM 为主；MLM 仍在检索、分类、句向量等场景常见。

---

## 五、预训练工程实践

- **数据配比**：不同来源（网页 / 书 / 代码）用 **采样权重** 控制；代码过高可能影响自然对话风格，反之则代码能力弱。
- **序列长度与打包**：在 **packing** 时把多段短文本拼成一条 max_length，提高 GPU 利用率（注意边界与 attention mask）。
- **稳定性**：学习率 warmup、梯度裁剪、损失 spike 监控；大集群上同步 **数据顺序与随机种子** 便于复现。
- **检查点**：按 step 保存 ckpt，记录 **数据版本、词表、清洗脚本 commit**；与 [2.4 分布式训练](../02_训练与微调/2.4_分布式训练.md) 中的并行策略协同。
- **评估**：除 loss / perplexity 外，可隔固定 step 跑 **下游 probe**（常识、代码补全等）观察是否训崩。
